Hi,

I've outlined the details for the TwitLonger API below. It's pretty simple, but if you have any questions give me a shout. One thing to note is that, as an open source client, you shouldn't be distributing your API keys with the source. I'm sure you realise that, but some people don't think about it and just hard code them rather than having a config variable.

Please make sure any implementation warns the user before submitting to TwitLonger, giving them the option to edit down to 140 characters if they want. Automatically posting encourages overuse of TwitLonger and many people don't realise they are sending their post to a third party site.

TwitLonger should NEVER be used when sending DMs as the site has no concept of direct messages. Twitter has blocked access to apps that send DMs to third party sites, so it would be a good idea to make sure you don't. The same also applies to private accounts.

Before an app goes live it would be great if you can send through about 100 words about the app, a decent resolution icon and a link to the application or developer site.

Twitlonger API
--------------------

For posting to Twitlonger:
----------------------------------
http://www.twitlonger.com/api_post
Request is made by POST (will throw a 405 error otherwise) and requires the following:

application     - reyntweets
api_key         - QKYsNt3hMonSgunD
username        - Twitter username (no password needed)
message - Body of message

If any of these are missing or the app/api_key are incorrect it will throw an error and set the appropriate http header response. Twitlonger is NOT checking if these are valid Twitter usernames, it is assumed that the application has already performed that part when the user registers with the app/service connecting to the api:

Optional:
in_reply                        - If the message is a reply, include the in_reply_to_status_id here
in_reply_user   - the username the message is a reply to

Errors look like this:
---------------------------
<twitlonger>
       <error>
               No Message Content
       </error>
</twitlonger>

If everything goes ok you should get back a 200 OK response and the data in the following format. Short is the short url I generate and this will be at the end of the post as the read more link. If shortening has failed this will come back as "isgdfail" and the message will have the full link to the Twitlonger post at the end. The content will always be 140 characters (unless you've sent a message that is significantly shorter, but it is assumed your app will only be accessing the service when the post is longer than 140). Note that Twitlonger does not handle posting to Twitter for third party apps, rather it returns the info that you then need to send to Twitter which makes error handling a lot easier on the client side and allows the appropriate app id to be set.

Successful Response:
------------------------------
<twitlonger>
       <post>
               <id>
                       8994dca4ac3680a7a40100d1b2457101
               </id>
               <link>
                       http://www.twitlonger.com/show/8994dca4ac3680a7a40100d1b2457101
               </link>
               <short>
                       http://is.gd/kr0z
               </short>
               <content>
                       This was a triumph, I'm making a note here, huge success. (cont) http://tl.gd/1234
               </content>
       </post>
</twitlonger>

ID CALLBACK
------------------------
Once a message has been successfully posted to Twitlonger and Twitter, it would be really useful to send back the Twitter ID for the message. This will allow users to manage their Twitlonger posts and delete not only the Twitlonger post, but also the Twitter post associated with it. It will also makes replies much more effective.

http://www.twitlonger.com/api_set_id
Call by POST
As with a message post, send your application name and api_key

Also send:
message_id (the id of the message you get back from Twitlonger when making a post)
twitter_id (the id of the message you've just posted to Twitter)

This should update the info in Twitlonger.

For reading
----------------
OK, technically to get it you go to:

http://www.twitlonger.com/api_read/message_id

but you can also take a raw twitlonger post

http://www.twitlonger.com/show/8994dca4ac3680a7a40100d1b2457101
and add
http://www.twitlonger.com/show/8994dca4ac3680a7a40100d1b2457101/fulltext

Which will do a header redirect to the api_read page.


If the message ID is invalid then you'll get back:

<twitlonger>
       <error>
               Invalid Message Requested
       </error>
</twitlonger>

If you want to do some integration of Twitlonger posts into the user timeline (rather than just loading the page in a webview) the site will be shortening links with the custom domain tl.gd in the form http://tl.gd/xxxx This will do a 301 redirect to the post and should make it easier to catch Twitlonger posts if you wish to. You could also grab the "xxxx" part on a tl.gd URL and grab the content of http://www.twitlonger.com/api_read/xxxx to avoid the header redirects.

There is also an option to get an oEmbed endpoint via embed.ly (http://embed.ly) which will return JSON for reading posts if that is easier for you.
